# 类人脑双系统全闭环AI架构 - 技术文档

## 一、架构概述

本架构基于阿里云Qwen3.5-0.8B作为唯一底座模型，完整实现了一套「海马体-新皮层双系统类人脑AI架构」，实现与人脑同源的"刷新即推理、推理即学习、学习即优化、记忆即锚点"的全闭环智能能力。

### 核心特性

1. **100Hz人脑级高刷新周期**：10ms刷新周期，对齐人脑gamma高频认知节律
2. **STDP学习机制**：脉冲时序依赖可塑性，无反向传播，纯本地时序信号驱动
3. **权重双轨拆分**：90%静态基础权重（永久冻结）+ 10%动态增量权重（可更新）
4. **海马体记忆系统**：完整的EC-DG-CA3-CA1-SWR生物对应实现
5. **三模式自闭环优化**：自生成、自博弈、自评判自动切换
6. **端侧部署优化**：INT4量化，显存≤420MB，适配安卓/树莓派

## 三、核心模块详解

### 模块1：Qwen3.5-0.8B底座模型基础改造

**权重双轨拆分**：
- 静态分支（90%）：继承官方预训练权重，永久冻结
- 动态分支（10%）：小权重随机初始化，STDP规则更新

**接口改造**：
- 特征输出接口：输出注意力特征、时序特征、语义特征
- 海马体门控接口：接收记忆锚点信号，引导注意力聚焦
- 角色适配接口：生成/验证/评判角色无缝切换

### 模块2：100Hz高刷新推理引擎

**单周期执行流程**（10ms内完成）：
1. 输入token接收与特征提取
2. 海马体记忆锚点调取与注意力门控加载
3. 窄窗口上下文+当前token的模型前向推理
4. 单周期输出结果生成
5. 全链路STDP权重本地刷新
6. 海马体情景记忆编码与更新
7. 全局工作记忆压缩更新

**窄窗口注意力**：
- 每周期仅处理1-2个token
- 注意力复杂度固定O(1)
- 动态聚焦，海马体引导

### 模块3：STDP时序可塑性权重系统

**核心规则**：
- LTP（长期增强）：前序激活早于后序，贡献度高 → 权重增强
- LTD（长期减弱）：时序错误或干扰 → 权重减弱

**更新节点**：
- 注意力层STDP：每周期更新
- FFN层STDP：每周期更新
- 自评判STDP：每10周期更新
- 海马体门控STDP：每周期更新

### 模块4：自闭环优化系统

**三种模式**：
1. **自生成组合输出**：并行生成2候选，STDP加权投票
2. **自博弈竞争优化**：提案-验证对抗迭代，直到收敛
3. **自双输出+自评判**：双候选生成，四维度评判选优

**自动切换**：基于输入关键词和任务难度自动选择模式

### 模块5：海马体记忆系统

**生物对应结构**：

| 生物脑结构 | 模型模块 | 核心功能 |
|-----------|---------|---------|
| 内嗅皮层EC | 特征编码单元 | 64维稀疏编码 |
| 齿状回DG | 模式分离单元 | 正交化投影，唯一记忆ID |
| CA3区 | 情景记忆库 | 模式补全，记忆召回 |
| CA1区 | 时序编码+门控 | 时间戳绑定，注意力控制 |
| 尖波涟漪SWR | 离线回放 | 记忆巩固，权重优化 |

**约束**：
- 情景记忆库最大2MB
- 单周期延迟≤1ms
- 最大召回锚点数：2

### 模块6：专项训练模块

**训练类型**：
1. 底座预适配微调：部署前一次性执行
2. 在线学习：推理过程中实时更新
3. 场景训练：针对特定场景专项优化

**约束**：仅更新10%动态权重，严禁触碰90%静态权重

### 模块7：测评体系

**测评维度**：
- 推理延迟：目标≤15ms
- 内存占用：目标≤420MB
- 准确率：目标≥80%
- 连贯性：目标≥85%
- 学习效率：目标≥70%
- 记忆召回准确率：目标≥85%
- 周期完成率：目标≥95%
